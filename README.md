# 🧩 Descripción General del Sistema

Este proyecto consiste en una arquitectura basada en **microservicios**, donde cada componente del sistema está desarrollado utilizando diferentes tecnologías y lenguajes de programación, simulando un entorno real empresarial con un enfoque **(multi-stack)**.

El objetivo principal es mostrar cómo múltiples servicios independientes pueden trabajar en conjunto para conformar una aplicación distribuida, manteniendo **desacoplamiento, escalabilidad y flexibilidad tecnológica**, así como su integración mediante contenedores utilizando Docker y su despliegue en entornos modernos, manteniendo simplicidad en cada microservicio y frontend.

Cada microservicio es responsable de un dominio específico del negocio, permitiendo que puedan ser desarrollados, desplegados y escalados de manera independiente. Además, el sistema incluye un frontend que consume estos servicios para ofrecer una experiencia unificada al usuario.

---

## 🧱 Microservicios del Sistema

1. **Users API** (.NET)  
2. **Products API** (Java)  
3. **Shipping API** (Python)  
4. **Inventory API** (Go)  
5. **Cart API** (Node.js)  
6. **Orders API** (.NET + PostgreSQL)  
7. **Payments API** (Java + PostgreSQL)  
8. **Reviews API** (Python + MongoDB)  
9. **Frontend** (Angular)  

---

## 🚀 Características Clave

- Arquitectura de microservicios desacoplada  
- Uso de múltiples lenguajes y tecnologías 
- Contenerización con Docker

# 🧩 System Overview

This project consists of a **microservices-based architecture**, where each system component is developed using different technologies and programming languages, simulating a real enterprise environment with a **(multi-stack)** approach.

The main objective is to demonstrate how multiple independent services can work together to form a distributed application, maintaining **decoupling, scalability, and technological flexibility**, as well as their integration through containers using Docker and deployment in modern environments, while keeping simplicity in each microservice and frontend.

Each microservice is responsible for a specific business domain, allowing them to be developed, deployed, and scaled independently. Additionally, the system includes a frontend that consumes these services to provide a unified user experience.
